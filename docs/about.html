About diagon

diagon makes a Monero marketplace stack easy to deploy and operate on a single Debian host, with I2P connectivity built in.

01Mission

Deliver a plug-and-play Monero marketplace stack for Debian that combines Diagon, Store, and Paywall over a locally running i2pd service. The project favors deterministic defaults, a single deployment profile, and generated operator documentation so that non-expert users can provision and run the full stack.

02Architecture

Diagon
Orchestrator and operator entrypoint; manages startup order, health checks, and config wiring.
Store
The marketplace frontend/backend component.
Paywall
Monero payment gating and settlement logic.
i2pd
Local router daemon that starts first and publishes the required I2P tunnels.

Component configs are generated from one deployment profile, secrets are injected from the environment or secure file paths, and health/readiness probes are exposed per component.

04Contributing

  • Read the roadmap to understand scope and phase.
  • Run go test ./... and validate profiles with diagonctl before opening a pull request.
  • Keep services local-only and secrets externalized, matching the existing contracts.
  • Open an issue to discuss significant changes first.
