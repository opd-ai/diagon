Installation guide

diagon targets a single Debian host running all components as local services. This guide covers the Debian baseline, the OpenBSD variant, and the interim compose fallback.

Hardware and platform requirements

Platform requirements for diagon
RequirementBaseline
Operating systemDebian 12 (bookworm), or OpenBSD for the variant pipeline
Deployment modelSingle host, local-only i2pd daemon
Core packagescurl, openssh-server, i2pd
Build toolchainGo (version pinned in go.mod / integration matrix)
NetworkingLocal interfaces plus i2pd tunnel endpoints; network namespace support for live probes
MoneroWallet/RPC endpoint (local or remote trusted node); stubbed mode available for testing

Pre-installation checklist

  • Confirmed your Debian baseline version matches the frozen integration matrix.
  • Installed the required packages (curl, openssh-server, i2pd).
  • Decided your Monero wallet strategy (local wallet, remote trusted node, or stubbed for testing).
  • Prepared secrets for injection via environment variables or secure file paths.
  • Cloned the repository and confirmed go test ./... passes.

Debian installation (baseline)

The Debian path is the primary, CI-proven deployment. At a high level:

  1. Install prerequisites

    sudo apt update
sudo apt install -y curl openssh-server i2pd

  2. Clone and validate

    git clone https://github.com/opd-ai/diagon.git
cd diagon
go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --format json

  3. Generate the Debian package baseline

    Emit the packaging plan (package layout, systemd units, post-install checks, uninstall/rollback semantics):

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --emit-debian-package-file /tmp/diagon-debian-package.json \
  --format json

  4. Confirm the dependency manifest is installable

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --integration-matrix-file .github/integration-matrix.json \
  --integration-environment debian-12 \
  --emit-debian-dependency-manifest-file /tmp/diagon-debian-dependencies.json \
  --format json

Service ordering on Debian

The generated systemd units enforce that diagon-i2pd.service starts before diagon-paywall.service, which starts before diagon-store.service, matching the frozen service contract.

OpenBSD variant installation

The OpenBSD variant reuses the same profile, validation policy, service contracts, bootstrap profile, and diagonctl tooling, but runs through a separate CI pipeline and produces a directly bootable raw disk image via OpenBSD autoinstall(8).

The variant is driven by a shared OpenBSD config set:

  • profiles/openbsd/install.conf — the autoinstall response file (the preseed analog).
  • profiles/openbsd/install.site — a pkg_add package hook run during install.
  • profiles/openbsd/pkg.list — the package list mirroring the Debian package intent.

See the Variants page for when to choose OpenBSD and how it differs from the Debian baseline.

Debian compose fallback (interim)

If full packaging is unavailable, diagon can generate a CI-validated compose/service bundle for Debian with documented manual install steps:

go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --integration-matrix-file .github/integration-matrix.json \
  --integration-environment debian-12 \
  --emit-debian-compose-bundle-file /tmp/diagon-compose-bundle.json \
  --format json

The bundle defines a pinned-image compose.yaml for i2pd, paywall, and store with dependency-aware startup ordering, a diagon-compose.service systemd unit for host-level lifecycle control, a secrets-aware environment template, and a manual install guide.

Post-installation verification

Run the live probes to confirm the stack is healthy:

go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --probe-live \
  --probe-timeout 45s \
  --probe-interval 750ms \
  --format json

Post-install checks confirm:

  • Services are enabled and started in the correct order.
  • Config files are present at their expected Debian paths.
  • Local health endpoints return a 2xx readiness response.
  • Expected i2pd tunnel listeners are reachable.
  • The aggregated health view reports all components ready.

Uninstall and rollback

Uninstall stops services in reverse order and preserves user data: /etc/diagon, /var/lib/diagon, and /var/log/diagon are kept, while only /run/diagon is removed.
