Debian · OpenBSD · Compose

Install

diagon targets a single host running every component as a local service. This page covers the Debian baseline, the OpenBSD variant, and the interim compose fallback.

01Requirements

Platform requirements
RequirementBaseline
Operating systemDebian 12 (bookworm), or OpenBSD for the variant
Deployment modelSingle host, local-only i2pd daemon
Core packagescurl, openssh-server, i2pd
Build toolchainGo (version pinned in go.mod)
NetworkingLocal interfaces plus i2pd tunnels; namespace support for live probes
MoneroWallet/RPC endpoint (local or remote trusted node); stubbed mode for testing

02Pre-install checklist

  • Debian baseline matches the frozen integration matrix.
  • Core packages installed (curl, openssh-server, i2pd).
  • Monero wallet strategy chosen (local, remote trusted node, or stubbed).
  • Secrets prepared for injection via environment or secure file paths.
  • Repository cloned and go test ./... passing.

03Debian (baseline)

The primary deployment path. At a high level:

  1. Install prerequisites

    sudo apt update
sudo apt install -y curl openssh-server i2pd

  2. Clone and validate

    git clone https://github.com/opd-ai/diagon.git
cd diagon
go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json --format json

  3. Generate the packaging plan

    Emit the package layout, systemd units, post-install checks, and rollback semantics:

    go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --emit-debian-package-file debian-package.json --format json

    See the emitted example on the Reports page.

Ordering

The generated systemd units enforce diagon-i2pd.service → diagon-paywall.service → diagon-store.service, matching the service contract.

04OpenBSD variant

The OpenBSD variant reuses the same profile, policy, service contract, bootstrap profile, and diagonctl tooling, but produces a directly bootable raw disk image via OpenBSD autoinstall(8). It is driven by a shared config set:

install.conf
The autoinstall response file — the preseed analog.
install.site
A pkg_add hook run during install.
pkg.list
The package list mirroring the Debian package intent.

See Variants for when to pick OpenBSD.

05Compose fallback (interim)

If full packaging isn't ready, generate a compose/service bundle for Debian with a manual install guide:

go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --integration-matrix-file .github/integration-matrix.json \
  --integration-environment debian-12 \
  --emit-debian-compose-bundle-file compose-bundle.json --format json

The bundle defines a pinned-image compose.yaml for i2pd, paywall, and store with dependency-aware ordering, a diagon-compose.service unit, an environment template, and a manual guide.

06Post-install verification

Run the live probes to confirm the stack is healthy:

go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --probe-live --probe-timeout 45s --probe-interval 750ms --format json
  • Services enabled and started in order.
  • Config files present at expected Debian paths.
  • Local health endpoints return 2xx.
  • Expected i2pd tunnel listeners reachable.
  • Aggregated health reports all components ready.
Uninstall

Uninstall stops services in reverse order and preserves user data: /etc/diagon, /var/lib/diagon, and /var/log/diagon are kept; only /run/diagon is removed.
