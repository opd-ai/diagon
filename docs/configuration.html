One profile, generated configs

Configure

diagon generates every component config from a single deployment profile. Secrets are injected from the environment or secure file paths — never hardcoded.

01File locations

Configuration files
FilePurpose
profiles/myprofile.packagesPackage list for the installer profile.
profiles/myprofile.preseedPreseed answers (e.g. passwd/root-login, time/zone).
profiles/validation-policy.jsonRequired packages and preseed keys.
profiles/service-contract.jsonListeners, health endpoints, ordering, i2pd tunnels.
profiles/local-single-host-bootstrap.jsonPhase 1 bootstrap defaults and startup order.
.github/integration-matrix.jsonFrozen Debian baseline, dependencies, and versions.
profiles/openbsd/OpenBSD variant config set.

02Policy format

*non-empty* is a sentinel meaning the key must exist and hold a non-empty value:

{
  "required_packages": ["curl", "i2pd", "openssh-server"],
  "required_preseed": {
    "passwd/root-login": "false",
    "time/zone": "*non-empty*"
  }
}

03Service contract

  • Required services. i2pd, store, and paywall must be defined.
  • Local-only hosts. Listeners and health endpoints must use localhost, 127.0.0.1, or ::1.
  • Startup ordering. i2pd before store and paywall; cycles are rejected.
  • Endpoint compatibility. Store → Paywall API link ports must match.
  • Tunnel types. client, http, http-proxy, socks, server; both Store and Paywall require mappings.

04Bootstrap defaults

  • Deterministic order: i2pd, paywall, store, diagonctl.
  • Local-only bind and health endpoints for all bootstrap components.
  • Default local Store → Paywall wiring and stubbed Paywall wallet mode.
  • Absolute Debian config/secret paths where file-backed secrets are used.

05Secrets

Never hardcode

Secrets come from environment variables or secure file paths. The only manual operator input beyond profile defaults should be secrets — for example a Monero wallet RPC endpoint.

For production wallet readiness, generate the wallet validation checklist (secrets preflight, Monero RPC probes, settlement verification):

go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --integration-matrix-file .github/integration-matrix.json \
  --integration-environment debian-12 \
  --emit-wallet-validation-checklist-file wallet-validation-checklist.md --format json

View a generated example on the Reports page.

06Config injection

Emit a bundle wiring Store, Paywall, and i2pd paths and ports:

go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --emit-config-injection-file config-injection.json --format json

07Live probe options

Live probe flags
FlagEffect
--probe-liveEnable listener, health, tunnel, and sequencing checks.
--probe-timeoutOverall readiness timeout (e.g. 45s).
--probe-intervalBase retry interval (e.g. 750ms); backoff is capped.

JSON output includes an aggregated_health view, marked failed when any component readiness probe fails.

08Tuning

  • Backoff pacing. Raise --probe-interval on slower hosts; capped exponential backoff avoids busy polling.
  • Timeouts. Increase --probe-timeout when i2pd tunnels are slow on first boot.
  • Ordering. Keep i2pd first to avoid readiness races.

09Troubleshooting

  • Wildcard package warning. A warning by default; --strict makes it an error.
  • Endpoint incompatibility. Confirm Store → Paywall ports match in service-contract.json.
  • Non-local host rejected. Use localhost, 127.0.0.1, or ::1.
  • Readiness timeout. A dependent service became ready before its dependency, or never became ready — check order and raise the timeout.
