Generated artifacts

Reports

diagon emits operational and compliance artifacts straight from your deployment profile. The files below are real outputs generated for the debian-12 environment — open them, or regenerate your own with diagonctl.

Regenerate

Every report is produced by a --emit-* flag on diagonctl. Each card lists the flag that creates it. To rebuild the whole set for your own profile, run the command in Regenerate all.

01Operations

Operator runbook

markdown · --emit-operator-runbook-file

Start, stop, status, logs, smoke-validation, and recovery commands for running Diagon, Store, Paywall, and i2pd on a single Debian host.

Open operator-runbook.md

Bootstrap quickstart

markdown · --emit-bootstrap-quickstart-file

One deterministic bootstrap profile and one secrets contract for local Debian bring-up, with explicit secret wiring and validation/probe commands.

Open bootstrap-quickstart.md

Production wallet validation checklist

markdown · --emit-wallet-validation-checklist-file

Secrets preflight, Monero wallet RPC probes, and paywall settlement verification steps for taking the stack from stubbed mode to production.

Open wallet-validation-checklist.md

04Regenerate all

All of the reports above were produced in one pass. Run this from the repository root to regenerate them into a directory of your choice:

go run ./cmd/diagonctl \
  --profile-dir profiles --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --integration-matrix-file .github/integration-matrix.json \
  --integration-environment debian-12 \
  --emit-operator-runbook-file out/operator-runbook.md \
  --emit-bootstrap-quickstart-file out/bootstrap-quickstart.md \
  --emit-wallet-validation-checklist-file out/wallet-validation-checklist.md \
  --emit-config-injection-file out/config-injection.json \
  --emit-debian-package-file out/debian-package.json \
  --emit-debian-dependency-manifest-file out/debian-dependency-manifest.json \
  --emit-debian-compose-bundle-file out/debian-compose-bundle.json \
  --emit-release-smoke-file out/release-smoke.json \
  --emit-release-baseline-file out/release-baseline.json \
  --emit-definition-of-done-file out/definition-of-done.json \
  --format json

Pass - as any emit path to print that report to stdout instead of a file.
