Getting started

Validate a deployment profile and boot the diagon stack on a single host. This quick start uses the diagonctl validation CLI shipped with the repository.

What you are building

A single Debian host running i2pd, Paywall, Store, and the Diagon orchestration check — wired from one profile, bound to local interfaces, and routed over I2P.

Prerequisites

Before you begin, confirm you have:

  • A supported host: Debian 12 (bookworm) or the OpenBSD variant.
  • Go installed (the repository targets the toolchain pinned in go.mod) to build and run diagonctl.
  • git to clone the repository.
  • Secrets you intend to inject (for example, a Monero wallet RPC endpoint) — never hardcoded, always supplied via environment or secure file paths.
  • Network namespace support if you plan to run live i2pd tunnels locally.

Quick start

  1. Clone the repository

    git clone https://github.com/opd-ai/diagon.git
cd diagon

  2. Validate the profile

    Run the validator against the bundled profile to check required packages, preseed keys, and warnings:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile

  3. Validate against the policy and service contract

    Add the JSON policy and service integration contract for a fuller check, with machine-readable output:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --service-contract-file profiles/service-contract.json \
  --format json

  4. Validate the single-host bootstrap profile

    Check the deterministic Phase 1 startup order and local default wiring:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --format json

  5. Run live readiness probes

    With the services running, verify listeners, health endpoints, tunnel reachability, and dependency sequencing:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --probe-live \
  --probe-timeout 45s \
  --probe-interval 750ms \
  --format json

First-run validation

A healthy first run should show, in JSON output:

  • All required packages present and required preseed keys satisfied.
  • Startup order resolved as i2pd → paywall → store → diagonctl.
  • When --probe-live is set, an aggregated_health object reporting all components ready.

Generate a quickstart guide

You can emit a deterministic onboarding guide tailored to your inputs with --emit-bootstrap-quickstart-file. Use - to print it to stdout.

Run the test suite

Confirm the tooling is healthy on your machine:

go test ./...

Strict mode

Treat warnings (such as wildcard package usage) as errors for CI-grade validation:

go run ./cmd/diagonctl --profile-dir profiles --profile-name myprofile --strict
