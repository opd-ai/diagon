Get started

Bring up the diagon stack on a single host and confirm it's healthy, using the diagonctl tool that ships with the repository.

01Prerequisites

  • A supported host: Debian 12 (bookworm), or the OpenBSD variant.
  • Go, matching the toolchain pinned in go.mod, to build and run diagonctl.
  • git to clone the repository.
  • Your secrets (for example a Monero wallet RPC endpoint), supplied via environment or secure file paths — never hardcoded.

02Quick start

  1. Clone the repository

    git clone https://github.com/opd-ai/diagon.git
cd diagon

  2. Validate the profile

    Check required packages, preseed keys, and warnings:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile

  3. Add the policy and service contract

    Validate against the JSON policy and the service topology, with machine-readable output:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --service-contract-file profiles/service-contract.json \
  --format json

  4. Validate the bootstrap profile

    Confirm the deterministic startup order and local default wiring:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --format json

  5. Probe live readiness

    With the services running, verify listeners, health endpoints, tunnel reachability, and dependency sequencing:

    go run ./cmd/diagonctl \
  --profile-dir profiles \
  --profile-name myprofile \
  --policy-file profiles/validation-policy.json \
  --bootstrap-profile-file profiles/local-single-host-bootstrap.json \
  --service-contract-file profiles/service-contract.json \
  --probe-live --probe-timeout 45s --probe-interval 750ms \
  --format json

03What a healthy first run looks like

  • All required packages present and required preseed keys satisfied.
  • Startup order resolved as i2pd → paywall → store → diagonctl.
  • With --probe-live, an aggregated_health object reporting every component ready.
Tip

Generate a runbook and quickstart tailored to your inputs and keep them next to your deployment — see Reports.

04Run the tests

Confirm the tooling is healthy on your machine:

go test ./...

05Strict mode

Promote warnings — such as wildcard package usage — into errors:

go run ./cmd/diagonctl --profile-dir profiles --profile-name myprofile --strict
