Monero · I2P · Debian · OpenBSD

diagon

A plug-and-play Monero marketplace stack for Debian that combines the Diagon orchestrator, Store, and Paywall over a locally running i2pd router — build, boot, and pass core integration checks with one profile.

Get started → View on GitHub

What is diagon?

diagon integrates four components into a single, single-host deployment that runs entirely on local interfaces and routes marketplace traffic through I2P:

Diagon

The system orchestrator and operator entrypoint. It wires component configs, manages startup order, and aggregates health checks.

Store

The marketplace frontend and backend, exposed over an i2pd tunnel and gated by the Paywall for settlement.

Paywall

Monero payment gating and settlement logic, with a stubbed wallet mode for CI and a production wallet validation path.

i2pd

A local I2P router daemon that starts first and publishes the tunnels the marketplace services depend on.

Key features

  • One-profile bootstrap. Boot the full stack from a single deployment profile with deterministic local defaults — no manual edits beyond secrets.
  • Deterministic startup. i2pd starts first, then Paywall, then Store, then the Diagon orchestration check.
  • Local-only by default. Every service binds to localhost, 127.0.0.1, or ::1; external reach goes through i2pd tunnels.
  • Validation CLI. diagonctl validates profiles, service contracts, i2pd tunnels, and runs live readiness probes.
  • Debian & OpenBSD. A Debian 12 (bookworm) baseline plus an OpenBSD variant that reuses the same profile and tooling.
  • CI-proven. An eight-stage CI pipeline proves the system can build, boot, and pass integration, contract, smoke, and packaging checks.
