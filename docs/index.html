Single-host marketplace stack

Run a private Monero marketplace on one Debian box.

diagon wires the Store marketplace, the Paywall settlement layer, and a local i2pd router from a single deployment profile — bound to local interfaces, reachable over I2P, and validated by one command.

$ go run ./cmd/diagonctl --profile-dir profiles --profile-name myprofile

Start here Install guide Browse reports

01What you get

  • One-profile bootstrap. Boot the whole stack from one profile with deterministic local defaults. The only manual input beyond defaults is your secrets.
  • Monero settlement, gated. Paywall handles payment gating and settlement, with a stubbed wallet mode for local bring-up and a production wallet checklist for going live.
  • Private by default. Every service binds to localhost; external reach travels only through i2pd I2P tunnels — nothing is exposed on a public interface.
  • Deterministic startup. i2pd comes up first, then Paywall, then Store, then the orchestration check — no readiness races.
  • Operator-ready. Generate a runbook, quickstart, and wallet checklist tailored to your profile so day-two operations are documented, not improvised.

02The pieces

Diagon
Orchestrator and operator entrypoint. Wires configs, orders startup, aggregates health.
Store
The marketplace frontend/backend, exposed through an i2pd tunnel.
Paywall
Monero payment gating and settlement, callable by Store over a local endpoint.
i2pd
Local I2P router that publishes the tunnels the marketplace depends on.

03It generates its own paperwork

diagon doesn't just boot — it emits the operational artifacts you need to run and audit the stack: an operator runbook, a bootstrap quickstart, a production wallet checklist, a Debian packaging plan, config-injection bundles, and more. Every report is generated from the same profile you deploy.
